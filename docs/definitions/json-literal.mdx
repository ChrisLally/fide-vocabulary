## Definition

A structured JSON object represented in canonical string form.

- **Boundaries:** Not a string.
- **Hex Code:** `a9`
- **Standard Alignment:** [`rdf:Literal`](https://www.w3.org/1999/02/22-rdf-syntax-ns#Literal) + [`rdf:JSON`](https://www.w3.org/1999/02/22-rdf-syntax-ns#JSON) (Exact fit)


## Fide ID Rules

These rules define how JSONLiteral can appear inside a [Fide ID](/fide-id).

### As Entity Type

- MUST use [Reference Type](/fide-id/specification/index#reference-type) [`JSONLiteral`](/vocabulary/definitions/json-literal) or [`NetworkResource`](/vocabulary/definitions/network-resource).




### As Reference Type

- [Reference](/fide-id/specification/index#reference) MUST use canonical JSON string form ([RFC 8785](https://www.rfc-editor.org/rfc/rfc8785)).

<details>
<summary>JSON references MUST be serialized in a stable canonical form before hashing.</summary>

- Valid [Reference](/fide-id/specification/index#reference): `{"a":1,"b":2}`
- Invalid [Reference](/fide-id/specification/index#reference): `{"b":2,"a":1}`
- Invalid [Reference](/fide-id/specification/index#reference): `{ "a": 1, "b": 2 }`
</details>
