### Graph Structure

Entity types that define the structure of the graph itself.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`Statement`](/vocabulary/definitions/statement) | `00` | A record of a Subject-Predicate-Object assertion. | Not a verified fact; verification is application-level. Not what it represents. |

### Agents

Entities with agency - they make decisions, take actions, and bear responsibility.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`Person`](/vocabulary/definitions/person) | `10` | A person (alive, dead, undead, or fictional). | Not an account or organization. |
| [`Organization`](/vocabulary/definitions/organization) | `11` | A structured collective of people or agents that acts as a unit. | Not a single person or software. Not a query-defined group (e.g. alumni by year); create only when the group itself needs to act. |
| [`SoftwareAgent`](/vocabulary/definitions/software-agent) | `12` | A running process that acts as an independent actor: bot, AI agent, smart contract. Can hold keys and sign. | Not a person or organization. Not static passive code (use CreativeWork for repos, images, logic). |

### Network Anchors

Network addresses where evidence lives: resolvable locations, platform handles, and cryptographic principals.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`NetworkResource`](/vocabulary/definitions/network-resource) | `20` | A resource identified primarily by addressability or resolution, such as a URL, URI, or CID. | Not a subset or account. Not an inhabitable spatial context (use Place). |
| [`PlatformAccount`](/vocabulary/definitions/platform-account) | `21` | An authority-based account principal (e.g., an email address or a specific GitHub, Google, or Mastodon user ID). The identity exists by the permission of a host system's database. | Authenticated by a host authority's database, not purely by mathematics. |
| [`CryptographicAccount`](/vocabulary/definitions/cryptographic-account) | `22` | A math-based account principal (e.g., an EVM address, Bitcoin wallet, or did:key). The identity exists mathematically and is authenticated via mathematical signature keys. | Authenticated via mathematical signature keys, independent of a host authority. |

### Knowledge

Things that represent intellectual property, abstract ideas, or data structures.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`CreativeWork`](/vocabulary/definitions/creative-work) | `30` | The most generic kind of creative work, including books, movies, photographs, software programs, etc. Non-expressive digital artifacts (binaries, model weights, dataset snapshots) are CreativeWork with refinement via Statements (artifactKind, mediaType, generatedBy, derivedFrom). | Not a raw value or formal concept. Not a running agent instance (use SoftwareAgent). Not a resource where location is primary identifier (use NetworkResource). |
| [`Concept`](/vocabulary/definitions/concept) | `31` | A formal concept, word, topic, category, or abstract idea defined in a taxonomy. | Not a plain raw value. |

### Spacetime

Entities bounded in time; optionally in physical or virtual space. Substrate-neutral Places, Events, Actions, and physical Objects.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`Place`](/vocabulary/definitions/place) | `40` | A spatial context that can contain presence, co-location, and interaction. Includes physical locations and persistent virtual environments (e.g., a concert venue, a Discord server or channel, a Zoom room, a VRChat instance). Physicality/virtuality is asserted via Statements, not the base type. | Not a happening or a physical artifact. Can an Actor have presence there? Is its identity a spatial address rather than a document location or account? |
| [`Event`](/vocabulary/definitions/event) | `41` | An occurrence or happening bounded in time. Can be observed/aggregated; may be caused by Actions, systems, or nature. Occurs at a Place (physical or virtual). | Not a specific exertion of agency (Action). |
| [`Action`](/vocabulary/definitions/action) | `42` | A discrete assertion or commitment attributable to an Agent. Often produces or records a change (e.g., a transaction, API call, signature). Can be linked to Events via Statements (causes, records, resultsIn). | Not a broad temporal happening (Event); must be driven by an Agent. |
| [`PhysicalObject`](/vocabulary/definitions/physical-object) | `43` | A tangible, inanimate physical object (e.g., hardware, vehicle, physical document, natural object). Physical only. Use Statements (madeBy, manufactured=true) if man-made vs natural matters. | Not a location (where). Not a digital artifact (use CreativeWork). |

### Literals

Typed scalar literals represented as first-class addressed values.

| Entity | Code | Definition | Not |
| :--- | :--- | :--- | :--- |
| [`TextLiteral`](/vocabulary/definitions/text-literal) | `a0` | A plain text sequence of characters. | Not a Concept. |
| [`IntegerLiteral`](/vocabulary/definitions/integer-literal) | `a1` | A whole number. | Not a decimal or string. |
| [`DecimalLiteral`](/vocabulary/definitions/decimal-literal) | `a2` | A decimal or floating-point number. | Not an integer. |
| [`BoolLiteral`](/vocabulary/definitions/bool-literal) | `a3` | A true or false value. | Not a number or string. |
| [`DateLiteral`](/vocabulary/definitions/date-literal) | `a4` | A calendar date (YYYY-MM-DD formatted according to ISO 8601). | Not a datetime or time. |
| [`TimeLiteral`](/vocabulary/definitions/time-literal) | `a5` | A time of day (formatted according to ISO 8601, represented in UTC). | Not a date or datetime. |
| [`DateTimeLiteral`](/vocabulary/definitions/date-time-literal) | `a6` | A specific point in time (ISO 8601 formatted, UTC normalized). | Not a date or time alone. |
| [`DurationLiteral`](/vocabulary/definitions/duration-literal) | `a7` | A calculated amount of elapsed time (ISO 8601 duration format). | Not a datetime. |
| [`URILiteral`](/vocabulary/definitions/uri-literal) | `a8` | A formalized Uniform Resource Identifier string. | Not plain text. |
| [`JSONLiteral`](/vocabulary/definitions/json-literal) | `a9` | A structured JSON object represented in canonical string form. | Not a string. |
